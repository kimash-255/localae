# Customer Screen List

This document lists customer-facing screens for product commerce, service booking, event participation, and post-purchase visibility.

## 1. Customer Screen Inventory

| # | Screen | Flow Area | Primary Purpose | Main Outcome |
| --- | --- | --- | --- | --- |
| C1 | Home / Explore Screen | Discovery | Main customer landing and discovery hub | User starts a path |
| C2 | Search Results Screen | Discovery | Show filtered products, services, and events | User narrows options |
| C3 | Product Detail Screen | Product Commerce | Explain product value and availability | User wants to buy |
| C4 | Service Detail Screen | Service Booking | Explain service value and conditions | User wants to book |
| C5 | Events Feed Screen | Events | Show active events and deal opportunities | User finds an event |
| C6 | Event Detail Screen | Events | Explain event details and offers | User wants to join or buy |
| C7 | Vendor Map / Nearby Sellers Screen | Matching | Show local sellers and distances | User sees available supply |
| C8 | Vendor Selection Screen | Matching | Compare vendors and fulfillment options | User selects a seller |
| C9 | Provider List Screen | Service Booking | Compare service providers | User selects provider |
| C10 | Service Mode Selection Screen | Service Booking | Choose delivery mode of service | Mode confirmed |
| C11 | Payment Flow Screen | Transaction | Confirm and pay for order | Transaction created |
| C12 | Service Request Screen | Transaction | Confirm and submit service booking | Request created |
| C13 | Participation Screen | Events | Join event, purchase, or interact | Participation created |
| C14 | Event Marketplace Screen | Events | Buy from vendors inside an event context | Event-linked order created |
| C15 | Order List Screen | Post-Purchase | Show all orders | User sees order history and active orders |
| C16 | Order Detail Screen | Post-Purchase | Show one order clearly | User understands exact order |
| C17 | Order Tracking Screen | Post-Purchase | Show order progress in real time | User sees status continuity |
| C18 | Service Status Screen | Post-Booking | Show service request progress | User sees service completion path |
| C19 | Event Tracking Screen | Post-Participation | Show event order or attendance progress | User sees event outcome |

## 2. Customer Flow Grouping

| Group | Screens |
| --- | --- |
| Discovery | C1, C2, C5 |

| Matching | C7, C8, C9, C10 |

## 3. Customer Screen Data Needs

| Screen | Key Data Needed |
| --- | --- |
| Home / Explore Screen | location, categories, featured items, nearby products, services, events |
| Search Results Screen | search query, filters, sort, availability, distance, price |
| Product Detail Screen | product info, images, pricing, stock, vendor options, reviews |
| Service Detail Screen | service description, price, mode options, provider summary, availability |
| Events Feed Screen | active events, timing, categories, offers |
| Event Detail Screen | event info, participation options, event vendors, schedule |
| Vendor Map / Nearby Sellers Screen | vendor locations, ETAs, open state, distance |
| Vendor Selection Screen | vendor comparison data, fulfillment mode, fees, ETA |
| Provider List Screen | provider profiles, price, rating, availability |
| Service Mode Selection Screen | available modes, timelines, cost implications |
| Payment Flow Screen | selected item, totals, fees, payment method, address or pickup option |
| Service Request Screen | selected provider, selected mode, timing, notes, price |
| Participation Screen | event ID, participation type, costs, selection state |
| Event Marketplace Screen | vendor listings, event offers, event-specific stock |
| Order Screens | order IDs, line items, statuses, vendor and rider references |
| Service Status Screen | service request ID, provider info, status timeline |
| Event Tracking Screen | event-linked order or participation status |

## 4. Customer Screen Notes

| Rule | Meaning |
| --- | --- |
| Discovery should never feel dead | Every major discovery screen needs a next action |
| Selection screens must reduce doubt | Vendor and provider choices should be easy to compare |
| Tracking is part of the product | Orders and services need visible continuity after payment or request |

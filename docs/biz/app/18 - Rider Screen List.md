# Rider Screen List

This document lists rider-facing screens for availability, job intake, navigation, completion, and earnings.

## 1. Rider Screen Inventory

| # | Screen | Flow Area | Primary Purpose | Main Outcome |
| --- | --- | --- | --- | --- |
| R1 | Rider Home Screen | Availability | Show rider landing state and quick status | Rider understands readiness |
| R2 | Availability Toggle Screen | Availability | Let rider go online or offline | Rider availability changes |
| R3 | Job Feed Screen | Job Intake | Show available jobs | Rider sees work options |
| R4 | Job Detail Screen | Job Intake | Explain one job clearly | Rider can decide to accept |
| R5 | Accept / Reject Screen | Job Intake | Confirm assignment action | Job is accepted or declined |
| R6 | Live Navigation Screen | Delivery | Show route execution in real time | Delivery progresses |
| R7 | Delivery Tracking Screen | Delivery | Show pickup and drop-off milestones | Status continuity |
| R8 | Completion Screen | Delivery | Confirm finished delivery | Job closed |
| R9 | Earnings Overview Screen | Earnings | Show current earnings snapshot | Rider sees value earned |
| R10 | Withdrawals Screen | Earnings | Start payout or withdrawal | Withdrawal requested |
| R11 | Earnings History Screen | Earnings | Show past jobs and money flows | Rider sees payout history |

## 2. Rider Flow Grouping

| Group | Screens |
| --- | --- |
| Availability | R1, R2 |
| Job Intake | R3, R4, R5 |
| Delivery | R6, R7, R8 |
| Earnings | R9, R10, R11 |

## 3. Rider Screen Data Needs

| Screen | Key Data Needed |
| --- | --- |
| Rider Home Screen | availability state, current assignment, quick stats |
| Availability Toggle Screen | online state, shift state, readiness checks |
| Job Feed Screen | job cards, pickup and drop-off area, payout estimate, distance, urgency |
| Job Detail Screen | route info, merchant info, customer info, package or order summary, payout breakdown |
| Accept / Reject Screen | assignment timeout, accept state, rejection reasons |
| Live Navigation Screen | route geometry, current location, pickup point, drop-off point, ETA |
| Delivery Tracking Screen | status milestones, proof checkpoints, communication shortcuts |
| Completion Screen | delivery confirmation, proof of delivery, notes, final status |
| Earnings Overview Screen | available balance, pending earnings, completed jobs summary |
| Withdrawals Screen | payout amount, payout method, withdrawal status |
| Earnings History Screen | completed payouts, payout timestamps, job-linked earning records |

## 4. Rider Screen Notes

| Rule | Meaning |
| --- | --- |
| Riders need fast decisions | Job screens should be highly scannable |
| Navigation is core, not secondary | Delivery flow depends on route clarity and status updates |
| Earnings must close the loop | Riders should immediately understand what work produced financially |
